# geo-command-center-platform

Full-stack GEO Command Center source snapshot: multi-tenant AI/search visibility SaaS with Next.js frontend, Supabase/Postgres/Drizzle data modeling, FastAPI backend services, worker flows, tests, provider/report pipelines, audits, citation tracking, and analytics dashboards.

## What It Shows

- Larger AI/data product architecture across frontend, backend, workers, scripts, tests, and database migrations.
- Multi-tenant dashboard and reporting workflows.
- Drizzle/Supabase/Postgres data modeling for brands, prompts, reports, citations, and visibility metrics.
- Backend services and worker-style processing around provider calls, audits, normalization, and reports.
- Practical AI product work: controlled workflows, persisted results, citations, reports, and user-facing review surfaces.

## Key Files

- `src` - Next.js frontend/application source.
- `backend` - backend service code.
- `worker` - async/background processing.
- `drizzle` - database schema/migrations.
- `supabase` - Supabase project assets.
- `tests` - test coverage/examples.
- `docker-compose.yml` - local service orchestration.

## Portfolio Notes

This is a sanitized source snapshot for portfolio review. The existing `geo-command-center` folder was left untouched; this full-stack source snapshot was added separately for comparison. Build outputs, dependency folders, local environment files, databases, credentials, large binary assets, archives, and generated outputs were intentionally excluded.
