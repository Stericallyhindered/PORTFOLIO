Additionally, the code has special cases for:

a (lowercase): Used to process a block of 48 bytes (though then ignored)
Z: Used to read a 12-byte VIN string
These identifiers match many of those seen in your logs. Note that some identifiers appearing in your logs (like W, N, X, = and others) aren’t handled in this block—those remain “unknown” in this code.

Usage: Processes a block of bytes to populate a table of values (typically RPM values at various points plus related sensor data)
Details:
Reads 24 bytes following the identifier and computes values for:
RPM values: rpm1500, rpm2000, …, rpm7000 (each calculated as (byteIN[offset]*256 + byteIN[offset+1]) / 10)
Other sensors: tmap_sensor, boost_limit_3rd, fuel values (for various RPMs), six_cyl_timing, and a series of CPS values (cps1500–cps7000)
Note: This block fills an entire calibration table or sensor mapping.
